# MCS-RTD 통합 플랫폼 공통 PRD

> 이 문서는 MCS 모듈(`prd-mcs.md`)과 RTD 모듈(`prd-rtd.md`)이 공유하는 공통 요소를 정의합니다.
> 각 모듈 PRD는 이 문서를 참조하며, 공통 기능은 여기서만 한 번 구현합니다.

---

## 🏗️ 모듈형 아키텍처

### 핵심 원칙

본 프로젝트는 **모듈형 독립 실행 아키텍처**를 채택합니다.

```
┌─────────────────────────────────────────────────────┐
│              공통 레이어 (Common)                       │
│   인증(Supabase Auth) · User 모델 · 공유 UI 컴포넌트     │
└──────────┬──────────────────────────┬────────────────┘
           │                          │
┌──────────┴──────────┐  ┌───────────┴───────────────┐
│   MCS 모듈           │  │   RTD 모듈                  │
│   (독립 실행 가능)     │  │   (독립 실행 가능)            │
│                      │  │                            │
│  • 레이아웃 모델러    │  │  • 노코드 룰 빌더            │
│  • 물류 실시간 모니터링│  │  • 룰 시뮬레이터             │
│  • 반송 제어 엔진     │  │  • 룰 실행 모니터링          │
│  • AI 경로 최적화     │  │  • 실행 통계                 │
│  • 시뮬레이션 검증    │  │  • LLM 룰 생성 보조(확장)    │
└──────────┬──────────┘  └───────────┬───────────────┘
           │                          │
           └──────────┬───────────────┘
                      │ REST API / 이벤트 버스
           ┌──────────┴──────────┐
           │   통합 연동 인터페이스  │
           │  (RTD↔MCS 이벤트)    │
           └─────────────────────┘
```

### 독립 실행 모드

| 실행 모드 | 설명 | 사용 시나리오 |
|----------|------|-------------|
| **MCS 단독 실행** | MCS 모듈만 기동. RTD 연동 없이 반송 제어·시뮬레이션·AI 경로 최적화 독립 사용 | MCS 경로 최적화 연구에 집중할 때 |
| **RTD 단독 실행** | RTD 모듈만 기동. MCS 연동 없이 룰 빌더·시뮬레이터·모니터링 독립 사용 | 디스패칭 룰 설계·검증에 집중할 때 |
| **통합 실행** | MCS + RTD 모듈 동시 기동. RTD 디스패칭 결과 → MCS 반송 명령 자동 생성 연동 | 전체 MCS-RTD 통합 시나리오 검증 시 |

### 모듈 간 통신

- 각 모듈은 **독립된 Next.js 애플리케이션**으로 실행 가능
- 모듈 간 통신은 **REST API 기반 이벤트 인터페이스**로 약결합(loose coupling) 유지
- 통합 실행 시에만 모듈 간 API 호출이 활성화되며, 단독 실행 시에는 연동 기능이 비활성 상태로 동작
- 공통 Supabase 프로젝트를 통해 인증 및 User 데이터를 공유

### 프로젝트 디렉토리 구조 (Turborepo 모노레포)

```
project-root/
├── apps/
│   ├── mcs/          # MCS 모듈 — Next.js 15 독립 앱
│   └── rtd/          # RTD 모듈 — Next.js 15 독립 앱
├── packages/
│   ├── ui/           # 공통 UI 컴포넌트 (shadcn/ui 기반)
│   ├── auth/         # 공통 Supabase Auth 모듈
│   └── types/        # 공통 TypeScript 타입 정의
├── turbo.json        # Turborepo 빌드 파이프라인 설정
└── package.json      # 루트 패키지 (워크스페이스 정의)
```

- **독립 실행**: `apps/mcs`, `apps/rtd` 각각 독립 빌드·배포 가능
- **공통 코드 공유**: `packages/` 패키지를 각 앱에서 import하여 코드 중복 제거
- **독립 배포**: 각 앱을 별도 Vercel 프로젝트로 독립 배포 가능

## 📄 공통 페이지별 상세 기능

### 로그인 페이지

> **구현 기능:** `FC001` | **인증:** 불필요 (공개 페이지)

| 항목 | 내용 |
|------|------|
| **역할** | 시스템 진입 인증 전용 페이지 |
| **진입 경로** | 비로그인 상태에서 보호된 페이지 접근 시 자동 리디렉션, 또는 헤더 로그인 클릭 |
| **사용자 행동** | 이메일/비밀번호 입력 → 로그인 클릭 |
| **주요 기능** | • 이메일/비밀번호 입력 폼<br>• 입력값 유효성 검사<br>• 회원가입 페이지 이동 링크<br>• **로그인** 버튼 |
| **다음 이동** | 성공 → 해당 모듈의 대시보드, 실패 → 에러 메시지 표시 |

---

### 회원가입 페이지

> **구현 기능:** `FC001` | **인증:** 불필요 (공개 페이지)

| 항목 | 내용 |
|------|------|
| **역할** | 신규 사용자 계정 생성 전용 페이지 |
| **진입 경로** | 로그인 페이지에서 회원가입 링크 클릭 |
| **사용자 행동** | 이메일/비밀번호/비밀번호 확인 입력 → 가입 완료 |
| **주요 기능** | • 이메일/비밀번호/비밀번호 확인 입력 폼<br>• 이메일 형식 및 비밀번호 강도 유효성 검사<br>• 로그인 페이지 이동 링크<br>• **회원가입** 버튼 |
| **다음 이동** | 성공 → 로그인 페이지, 실패 → 에러 메시지 표시 |

## 🛠️ 공통 기술 스택 (최신 버전)

> 각 모듈은 아래 공통 스택을 기반으로 하며, 모듈별 추가 스택은 각 PRD에서 정의합니다.

### 🎨 프론트엔드 프레임워크

- **Next.js 15** (App Router) - React 풀스택 프레임워크
- **TypeScript 5.6+** - 타입 안전성 보장
- **React 19** - UI 라이브러리 (최신 동시성 기능)

### 🎨 스타일링 & UI

- **TailwindCSS v4** (설정파일 없는 새로운 엔진) - 유틸리티 CSS 프레임워크
- **shadcn/ui** - 고품질 React 컴포넌트 라이브러리
- **Lucide React** - 아이콘 라이브러리

### 📝 폼 & 검증

- **React Hook Form 7.x** - 폼 상태 관리
- **Zod** - 스키마 검증 라이브러리

### 🗄️ 인증 & 데이터베이스

- **Supabase** - BaaS (인증, 데이터베이스, 실시간 구독)
- **PostgreSQL** - 관계형 데이터베이스 (Supabase 포함)

### 🚀 배포 & 호스팅

- **Vercel** - Next.js 15 최적화 배포 플랫폼

### 📦 패키지 관리

- **npm** - 의존성 관리
